Sparse Gaussian Process¶

Covariance Function I'm using the product of two squared exponentials. One for the time of day and one for the time of year. The squared exponential seems to be a quasi default kernel for Gaussian Processes, therefore I just went with it.

Hyperparameters The squared exponential kernel has two parameters: lengthscale and output variance. I chose lengthscale to be 100. This is purely a choice based on observation. 100 seems to be an overlapping value that works well for both minutes with a range [0,1440] and days [0,365] when I specify a kernel based on only one of the input variables. For output variance I chose 0.1, which seemed like a reasonable value for temperatures.

The Sparse Gaussian Process also takes a parameter n_inducing, which refers to the number of inducing variables (or pseudo inputs) for our GP. I will use GPy's in-built methods to optimize for where to take inducing inputs from.

'''
Auxilliary functions to Ridge Regression.

1. Split data at a specified cutoff point.
2. Apply the rbf transform on data given parameter values
3. Fit a Ridge Regression model on the data and evaluate model by MSE
4. Create a center array for x_days based on n_rbf - range(0,365)
5. Create a center array for x_minutes based on n_rbf - range (0,1435)

Index 525887 stands for the split between 2015 and 2016.

'''
cutoff=525887
def split_data(df,cutoff):
    x_train=[df.days.values[:cutoff],df.minutes.values[:cutoff]]
    x_test=[df.days.values[cutoff:],df.minutes.values[cutoff:]]
    y_train=df.temp.values[:cutoff]
    y_test=df.temp.values[cutoff:]
    return (x_train,x_test,y_train,y_test)

def rbf_transform(df,cutoff,center1,center2,sigma1,sigma2,alpha):
    data=split_data(df,cutoff)
    x1=np.array(rbf_kernel(data[0][0].reshape(-1,1),center1,gamma=1/sigma1))
    x2=np.array(rbf_kernel(data[0][1].reshape(-1,1),center2,gamma=1/sigma2))
    x_train=np.concatenate((x1,x2),axis=1)
    
    x3=np.array(rbf_kernel(data[1][0].reshape(-1,1),center1,gamma=1/sigma1))
    x4=np.array(rbf_kernel(data[1][1].reshape(-1,1),center2,gamma=1/sigma2))
    x_test=np.concatenate((x3,x4),axis=1)
    
    return x_train,x_test,data[2],data[3]


def regr(df,cutoff,center1,center2,sigma1,sigma2,alpha):
    x_train,x_test,y_train,y_test=rbf_transform(df,cutoff,center1,center2,sigma1,sigma2,alpha)
    regr_rbf=linear_model.Ridge(alpha=alpha,fit_intercept=True) #model selection
    regr_rbf.fit(x_train,y_train) #model fitting
    y_pred=regr_rbf.predict(x_train) #model prediction
    MSE=metrics.mean_squared_error(y_train, y_pred) #calculate MSE train for model
    return MSE

def center1(n_rbf):
    #function to allocate centers based on different number of RBFs to be specified
    return np.linspace(0,365,n_rbf).reshape(-1,1)

def center2(n_rbf):
    #function to allocate centers based on different number of RBFs to be specified
    return np.linspace(0,1435,n_rbf).reshape(-1,1)

Conclusion¶

I found the setup and execution of the Sparse GP a lot more straightforward than the RBF Ridge Regression we did for our previous assignment. If I had more practice with the choice of kernels and optimization methods, I would definitely choose this method over the previous one. I also think, that Sparse GP will reliably produce better results given how little we actually need to bother fine-tuning parameter values.

Reading about Sparse GP, I found that this technique was developed to cut down on the exponential time complexity of the original GP method. However, the Sparse GP method does not affect memory usage, as the GPy.SparseGPRegression was using exactly $N^2$, which I assume will create a lot of issues for bigger datasets.
