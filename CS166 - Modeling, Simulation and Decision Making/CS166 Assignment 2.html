import numpy as np
from matplotlib import pyplot as plt
class TrafficSimulation:

    def __init__(self, length=100, density=0.3, p=0.5, max_speed=5):
        self.road_length = length #size
        self.density = density #initial car density
        self.max_speed = max_speed #cap on speed
        self.p = p #prob cars will slow down at a update
        self.flow=0 #flow counter

        ind_rand = np.random.choice(range(self.road_length)
                                    ,size=int(round(self.density * self.road_length)), replace=False)
        self.state = -np.ones(self.road_length, dtype=int)  # -1 means empty cell
        self.state[ind_rand] = np.random.randint(0, self.max_speed + 1, size=len(ind_rand))

    def update(self, show=True):
        for i in range(self.road_length):
            if self.state[i] != -1: #if car
                dist = 1
                while self.state[(i + dist) % self.road_length] == -1:
                    dist += 1
                #accelerate
                if self.state[i] < self.max_speed:
                    self.state[i] += 1
                #slow down
                if self.state[i] >= dist:
                    self.state[i] = dist -1
                #random slow down
                if (self.state[i] > 0) and (np.random.uniform() < self.p):
                    self.state[i] -= 1
                #count flow
                if self.state[i]+i>=self.road_length:
                    self.flow+=1

        if show:
            self.show()
        #create and write new state
        new_state = -np.ones(self.road_length, dtype=int)
        for i in range(self.road_length):
            if self.state[i] != -1:
                new_state[(i + self.state[i]) % self.road_length] = self.state[i]
        self.state = new_state

    def show(self):
        print(''.join('.' if x == -1 else str(x) for x in self.state))

How do traffic jams form?¶

Traffic jams are the consequence of the stochastic process of slowing-down. Parameter $p$ determines the probability with which a single car might decrease its velocity by 1. When this happens, cars behind the slowing down car need to slow down, initiating a chain reaction of slowing down in cars further down.

Traffic jams may be identified on the above visualisations as a string of zeros, indicating motionless cars. Jams travel in shockwaves in a backwards direction. This can be best seen on the visualisation at $density=0.5$.

Two-lane Highway¶

Assumptions¶

The two-lane model is an extension of the single lane model, with the additional feature of cars being able to change lanes. The update rules for moving forward stayed the same, and additional rules for changing lanes were added.

I assumed that (unlike in real life) there are no such rules as "keeping right" or "keeping left". Thus, lane changes only happen so for cars to avoid jams. Also, cars in both lanes are going in the same direction.

Parameters¶

The parameters of the model stayed the same as for the single model. The simulation can still be initialised to different road lengths, car densities, max speeds, and probabilities of slow down.

In contrast to the single-lane model, keeping track of the traffic flow has changed. Instead of measuring the total flow, an average of flow is calculated between left and right lanes, so as to give us results that are comparable to single-lane traffic results.

Update Rules¶

As the update rules for moving forward stayed the same as in the single lane model, the only additional update rules concerned the changing of lanes. The rules to this update are (for further details, see comments in code):

  1. Check if car ahead in the same lane would necessitate slow-down. If yes, proceed.
  2. Check if car ahead in the next lane would necessitate slow-down. If no, proceed.
  3. Check if changing lanes would necessitate cars in the other lane to slow down. If no, proceed.
  4. Change Lanes.

The simulation is carried out in two steps. First, the lane changes are actions and then the original forward moving rules come. While, it might seem counterintuitive to split the process into two steps, but in effect the results are the same.

    def update_right(self,lane):
        #function carrying out rules from Part 1
        for i in range(self.length):
            if lane[i] != -1: #if car, i.e. not empty
                dist = 1
                while lane[(i + dist) % self.length] == -1:
                    dist += 1
                if lane[i] < self.max_speed:
                    lane[i] += 1
                if lane[i] >= dist:
                    lane[i] = dist - 1
                if (lane[i] > 0) and (np.random.uniform() < self.prob_slow):
                    lane[i] -= 1

        new_state = -np.ones(self.length, dtype=int)
        for i in range(self.length):
            if lane[i] != -1:
                new_state[(i + lane[i]) % self.length] = lane[i]
        self.state_right = new_state

                        #check if car behind in other lane has speed lower than where car would move
                        index=-1
                        while self.state_right[(i + index) % self.length] == -1:
                            index-=1
                        if self.state_right[i+index]-abs(index)<self.state_middle[i]:
                            #move car to other lane
                            self.state_right[i]=self.state_middle[i]
                            self.state_middle[i]=-1     
        
        #update both lanes according rules of one-lane simulation
        self.update_left(self.state_left)
        self.update_middle(self.state_middle)
        self.update_right(self.state_right)
        
        #update flow
        for k in range(5):
            if self.state_left[-k]+(self.length-k)>self.length:
                self.flow+=1
            if self.state_right[-k]+(self.length-k)>self.length:
                self.flow+=1
            if self.state_middle[-k]+(self.length-k)>self.length:
                self.flow+=1        
        
        #display
        if show:
            self.show()
        
    def show(self):
        print(''.join('.' if x == -1 else str(x) for x in self.state_left))
        print(''.join('.' if x == -1 else str(x) for x in self.state_middle))
        print(''.join('.' if x == -1 else str(x) for x in self.state_right))
        print('')

How applicable is this model to traffic in Berlin?¶

The multi-lane traffic models were constructed to be symmetric. That is, they don't follow a rule of "keeping right" or "keeping left". In Berlin, according to German traffic law, cars in unidirectional, multi-lane roads must keep to the right. That is, unless they want to overtake, they must be travelling in the right-most lane of the road. From this perspective it is true that this model does not realistically represent Berlin traffic.

However, in other ways it does. Most importantly, the patterns observed of traffic flow at different densities are similar to what I see day-to-day as I cycle. Especially that high densities of cars, eventually end up causing a traffic jam without there being an obstacle in the way to explain it. Thus the model can realistically represent aspects of human behaviour when driving.
